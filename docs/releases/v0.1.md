# v0.1-k3s-gitops-observability

First platform milestone for **py_wallet**. This version demonstrates a
production-like DevOps/SRE lab running on k3s, deployed via GitOps with
observability in place.

## Highlights

- FastAPI/PostgreSQL application
- CI/CD with GHCR image publishing by SHA
- GitOps deployment through Argo CD (app-of-apps, pull-based)
- PreSync Alembic migrations (same image SHA as the rollout)
- TLS through cert-manager + Let's Encrypt
- Kubernetes probes and resource limits
- Dedicated ServiceAccount (no CI cluster access)
- Prometheus/Grafana monitoring (kube-prometheus-stack)
- ServiceMonitor-based app scraping
- Basic Grafana dashboard as code

## Scope

| Repo | Role in this release |
|------|----------------------|
| [py_wallet](https://github.com/amysyutin/py_wallet) | Application source, tests, Docker image, CI build & push to GHCR |
| py_wallet-infra (this repo) | Kubernetes manifests, Argo CD apps, GitOps deploy, monitoring |

## Known gaps / next steps

Tracked in the [Production Readiness Status](../../README.md#production-readiness-status):

- Secrets: move from out-of-band Kubernetes Secrets to SOPS/SealedSecrets
- Security hardening: NetworkPolicy, securityContext
- Backup/restore: pg_dump CronJob + restore runbook
- CI/CD: image signing / SBOM
- Monitoring: alerts / SLOs
